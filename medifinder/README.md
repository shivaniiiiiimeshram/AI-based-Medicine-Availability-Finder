# MediFinder — AI-Based Medicine Availability Finder

A full-stack web application that lets users search for medicines at nearby pharmacies, make reservations, and scan medicine labels using OCR. Pharmacists can register, manage their inventory, and handle reservations from a dedicated dashboard.

---

## Tech Stack

| Layer | Technology |
|---|---|
| Frontend | HTML, CSS, Vanilla JavaScript |
| Backend | Node.js + Express |
| Database | MongoDB (via Mongoose) |
| ML / OCR | Python + Flask (Naive Bayes + Tesseract OCR) |

## Prerequisites

Make sure these are installed on your machine before starting:

- **Node.js** v18 or newer — https://nodejs.org
- **Python** 3.8 or newer — https://python.org
- **MongoDB** — either:
  - [MongoDB Atlas](https://www.mongodb.com/cloud/atlas) (free cloud cluster), **or**
  - Local MongoDB: https://www.mongodb.com/try/download/community
- **Tesseract OCR** (required for the label-scanning feature):
  - **Windows:** Download installer from https://github.com/UB-Mannheim/tesseract/wiki
  - **macOS:** `brew install tesseract`
  - **Linux (Ubuntu/Debian):** `sudo apt-get install tesseract-ocr`

## Setup

### 1. Clone / download the project

```bash
git clone <repo-url>
cd medifinder
```

### 2. Configure environment variables

Copy the example file and fill in your values:

```bash
cp .env.example .env
```

Edit `.env`:

```env
MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/medifinder
SESSION_SECRET=any-long-random-string
PORT=3000
```

- `MONGODB_URI` — your MongoDB connection string (Atlas or `mongodb://localhost:27017/medifinder` for local)
- `SESSION_SECRET` — any random string (e.g. `openssl rand -hex 32`)
- `PORT` — defaults to `3000` if not set

## Running the Project

You need **three terminal windows** — one per service.

### Terminal 1 — Node.js Backend

```bash
# Install dependencies (first time only)
npm install

# Run in development mode (auto-restarts on file changes)
npm run dev

# Or run without nodemon
npm start
```

The backend starts at: **http://localhost:3000**  
Open the app at: **http://localhost:3000/medifinder/**

### Terminal 2 — Python ML API

```bash
cd ml-api

# Create a virtual environment (recommended)
python -m venv venv

# Activate it
# Windows:
venv\Scripts\activate
# macOS / Linux:
source venv/bin/activate

# Install Python dependencies (first time only)
pip install -r requirements.txt

# (Optional) Retrain the model if needed
# python train.py

# Start the Flask server
python app.py
```

The Python ML API starts at: **http://localhost:5000**

### Terminal 3 — (Optional) Seed sample data

If you want to populate the database with 5 sample pharmacies and medicines in Kuala Lumpur:

```bash
npm run seed
```

## Features

### User-facing
- Search medicines by name with fuzzy suggestions and distance-based results
- Symptom-to-medicine category fallback (ML-powered)
- Scan a medicine label photo — OCR extracts the name and searches automatically
- Geolocation-aware results (sorted by distance)
- Reserve medicines at specific pharmacies
- View active and completed reservation history
- In-app notifications when reservation status changes

### Pharmacist Portal
- Register with full pharmacy profile (name, address, contact, location)
- Add and delete medicines from inventory
- Manage reservations: Pending → Confirmed → Completed
- View historical completed reservations

## API Endpoints

### Backend (Node.js — port 3000)

| Method | Path | Description |
|---|---|---|
| POST | `/medifinder/auth/register` | Pharmacist registration |
| POST | `/medifinder/auth/login` | Pharmacist login |
| POST | `/medifinder/auth/logout` | Pharmacist logout |
| GET | `/medifinder/auth/me` | Get current pharmacist session |
| POST | `/medifinder/user/register` | User registration |
| POST | `/medifinder/user/login` | User login |

| PATCH | `/medifinder/reservation/:id/status` | Update reservation status |

| Method | Path | Description |
|---|---|---|
| GET | `/` | Health check |
| POST | `/predict` | Predict medicine from symptom text |
| POST | `/scan` | Extract text from medicine label image |

## Common Issues

**"OCR not available" warning**  
Tesseract is not installed or not in your PATH. Install it (see Prerequisites) and restart the Python server.

**MongoDB connection error**  
Check your `MONGODB_URI` in `.env`. For Atlas, ensure your IP is whitelisted in Network Access settings.

**Port already in use**  
Change the `PORT` in `.env` for the backend. For the Flask API, edit the port in `ml-api/app.py` line `app.run(..., port=5000)` and update `ML_API` in `routes/scan.js`.

**"model.pkl not found"**  
The model files are included. If they go missing, regenerate them:
```bash
cd ml-api
python train.py
```
